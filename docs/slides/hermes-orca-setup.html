Hermes + Orca

Setup paths: bridge (HTTP tools) vs in-app Hermes tile (Responses API). This deck is a visual companion to docs/hermes-orca-visual-setup.md.

Two main paths

Bridge

Hermes / Pi / OpenClaw calls GET /api/canvas/tools and POST /api/canvas/execute. Keep the Orca UI open so uiClients ≥ 1.

In-app tile

Install hermes CLI, run API_SERVER_ENABLED=true hermes gateway, align Integrations base URL, add a hermes_agent tile.

1 — Hermes CLI

Install from NousResearch, then verify in a normal terminal:

hermes --version

If this fails, fix PATH before relying on gateway autostart from Orca.

2 — Gateway

API_SERVER_ENABLED=true hermes gateway

Default API often listens on 127.0.0.1:8642. Set API_SERVER_KEY in ~/.hermes/.env if required; Orca can auto-read it when the Integrations key is empty.

3 — Orca settings

  • Integrations → Hermes API — base URL, model, optional key override.
  • Agent → Hermes — show Hermes tile in menus; use Run diagnose if things break.

Bridge smoke

curl -s http://127.0.0.1:3001/api/canvas/bridge-status

Expect uiClients ≥ 1 with the app window open. Add Authorization: Bearer … when CANVAS_BRIDGE_TOKEN is set.

Skills from Orca repo

npm run hermes:install-skills

Optional: HERMES_SKILLS_DIR=… for a non-default Hermes skills directory.

When something breaks

  • Orchestrator tool: diagnose_hermes_setup
  • Settings: Run diagnose under Agent → Hermes
  • Turn off Show Hermes agent tile if you do not use Hermes — fewer confusing tool offers.
