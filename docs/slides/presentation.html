Orca Coder

Infinite canvas IDE for multi-agent coding

Pan, zoom, and arrange agents, terminals, editors, and previews on one surface — with an orchestrator that drives tools against your repo.

Swiss Modern Editorial grid · red rule · Archivo + Nunito

Positioning

One workspace, many agents

Run Claude, Codex, Gemini, and your own flows in parallel without losing context — tiles keep outputs and files visually separated.

Canvas

Tiles, not tabs

  • Infinite pan and zoom; focus mode for deep work
  • Z-order, resize presets, and mission-control style overview
  • Orchestrator reveals the active module as tools run
Model Each tile has id, title, position, size, and meta — the orchestrator lists and updates them safely.

Modules

Six core tile types

Agent Live model output
Terminal PTY + xterm.js
Editor Monaco
Browser Local / web preview
Diff Writes & review
Todo Tasks & handoff

Orchestrator

Tool loop on your repo

  • Read / write files, list directories, workspace switch
  • Create and update canvas modules from chat
  • Parallel tool batches when paths don’t conflict
Quality Large tool results can be compressed before they hit the model — less context blow-up on long sessions.

Integrations

Bring your own agent

External runners (Hermes, Pi, OpenClaude-style adapters) can call the same tool contract over HTTP/WebSocket — the canvas executes tools while your UI stays in sync. GitHub research can use a dedicated GitHub (gh) module instead of fragile iframes.

Stack

Implementation

  • React 18, TypeScript, Vite, Tailwind, Zustand
  • Companion server: Node, Express, WebSocket
  • Desktop: Tauri + native shell where needed
Dev npm install && npm run dev

Workflow

Keyboard-first

  • ⌘1–⌘6 — add Agent, Terminal, Browser, Todo, Editor, Diff
  • ⌘Enter — focus mode · Esc — exit · ⌘0 — reset view
